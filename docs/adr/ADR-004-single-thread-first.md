# ADR-004: Phase 1〜5 は単一スレッド、Phase 6 で Rayon 並列化

- **日付**: 2026-06-20
- **状態**: Accepted

## 背景

Rayon を最初から入れると、プランナー・エグゼキューターの設計がスレッドセーフ制約に縛られて複雑になる。

## 決定

Phase 1〜5 は単一スレッドで実装し、Phase 6 で `PhysicalPlan` に並列ノードを追加して Rayon を導入する。

## 理由

- 単一スレッドで正しく動く実装を先に作ることで、並列化後のバグとの切り分けが容易
- Arrow RecordBatch は `Send + Sync` なので、後から Rayon を追加するコストは低い
- Criterion ベンチマークで before/after を比較し、並列化の効果を数値で確認できる

## トレードオフ

- Phase 5 までは大規模データに対してスループットが低い
