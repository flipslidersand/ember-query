# ADR-001: 実装言語に Rust を選ぶ

- **日付**: 2026-06-20
- **状態**: Accepted

## 背景

SQL エンジンはメモリ管理・SIMD・列指向処理が性能に直結する。
このプロジェクトは「処理系実装を通じた Rust 習得」が学習目的の一つ。

## 決定

Rust を使う。

## 理由

- Arrow / Parquet の公式クレートが Rust ネイティブ (DataFusion と同一エコシステム)
- GC なし・ゼロコストアブストラクションで列指向処理の性能を引き出せる
- 所有権モデルにより AST ツリー操作・プラン変換を安全に実装できる
- 将来的に Wasm コンポーネントへのコンパイルも視野に入る

## トレードオフ

- Python より開発速度は遅い（型エラーを全部コンパイル時に直す必要がある）
- 初期学習コストが高い（借用チェッカー）
