# ADR-002: パーサーに pest (PEG) を選ぶ

- **日付**: 2026-06-20
- **状態**: Accepted

## 背景

SQL パーサーの実装方法として、手書き再帰下降・PEG (pest)・パーサーコンビネータ (nom) の選択肢がある。

## 決定

`pest` (PEG 文法) を使う。

## 理由

- 文法を `.pest` ファイルに宣言的に書けるため、SQL サブセットの全体像を把握しやすい
- 演算子優先順位を `PRATT_PARSER` で自然に表現できる
- nom と比べてエラーメッセージが読みやすく、デバッグが容易
- 手書き再帰下降は柔軟だが、文法変更のたびに大規模な書き換えが発生する

## トレードオフ

- nom より実行速度がやや遅い（プロトタイプ規模では無視できる）
- 左再帰が書けないため、一部の文法パターンで迂回が必要
